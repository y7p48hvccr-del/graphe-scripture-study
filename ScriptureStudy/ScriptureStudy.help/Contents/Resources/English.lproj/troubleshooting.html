Troubleshooting

Solutions to common issues.

No modules appear in the Library

  • Check that you have selected a modules folder — open the Library tab and look for the folder path at the bottom.
  • Confirm your .SQLite3 files are directly inside the selected folder (not in a subfolder).
  • Click the Refresh button (↺) in the Library toolbar to re-scan.

Strong's words are not highlighted

  • Confirm your Bible module includes Strong's tagging — not all translations do.
  • Confirm you have a Strong's lexicon installed and selected in the Companion panel Dictionaries tab.
  • Some modules use non-standard Strong's markup that may not be fully supported.

The dictionary lookup is not working

  • Confirm a dictionary module is installed and selected in the Companion panel Dictionaries tab.
  • The clipboard monitor checks every 0.5 seconds — copy the word again and wait a moment.
  • Some dictionary modules use topic keys that differ from standard English spelling — try a different dictionary.

e-Sword import fails

  • Encrypted commercial modules cannot be converted — you will see a clear error message.
  • Ensure Python 3 is installed on your Mac (it is included by default on macOS).
  • Check that your modules folder is writable.

Notes are not saving

  • Check that ~/Documents/ScriptureStudy Notes/ exists and is writable.
  • Notes save automatically within one second of you stopping typing — you do not need to do anything.

AI summaries are not appearing

  • For Claude: check your API key in Settings and ensure you have available API credits.
  • For Ollama: confirm Ollama is running (ollama serve in Terminal) and the model name matches exactly.

The app is showing the wrong colour scheme

ScriptureStudy respects both its own theme setting and macOS's light/dark mode. If colours look unexpected, check Settings → Appearance and ensure your chosen theme matches your macOS appearance preference.
