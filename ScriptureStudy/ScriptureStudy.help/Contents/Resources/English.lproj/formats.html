Module Formats

Technical details about the file formats ScriptureStudy understands.

MyBible SQLite3

The primary format. Each module is a single .SQLite3 file containing an info table with metadata and one or more content tables depending on the module type. ScriptureStudy detects the module type from the tables present and the is_strong field in the info table.

e-Sword / MySword

ScriptureStudy can import these formats and convert them to MyBible SQLite3. The converter handles Strong's markup (<WG> and <WH> tags) translating them to the MyBible <S> format. Encrypted commercial e-Sword modules cannot be converted.

Note files

Notes are stored as plain UTF-8 text files in ~/Documents/ScriptureStudy Notes/. The format is:

Title
BookNumber
ChapterNumber
VerseNumbers (comma-separated)
ISO 8601 date
UUID
locked/unlocked
---
Note content begins here...

These files can be opened, edited and backed up with any text editor or file manager.
