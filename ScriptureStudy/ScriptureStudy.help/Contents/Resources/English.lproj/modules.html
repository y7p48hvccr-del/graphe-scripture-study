Installing Modules

ScriptureStudy uses the open MyBible module format — a SQLite3 database with a standardised schema.

Module types

TypeFile indicatorDescription
BibleContains a verses tableA complete Bible translation
CommentaryContains a commentaries tableVerse-by-verse commentary
Strong's lexiconis_strong = true in info tableHebrew or Greek lexicon keyed to Strong's numbers
DictionaryContains a dictionary table, is_strong = falseGeneral dictionary (e.g. Webster's)
OtherAnything elseDevotionals, topic lists, etc.

Adding modules

  1. Download a .SQLite3 module file from the MyBible community or any compatible source.
  2. Place it in your chosen modules folder.
  3. In the Library tab, click the Refresh button (↺) in the toolbar.
  4. The module appears in the appropriate section automatically.

Importing e-Sword and MySword modules

ScriptureStudy can convert modules from e-Sword and MySword formats. Click the Import e-Sword button in the Library toolbar and select a file with one of these extensions:

  • .bblx — e-Sword Bible
  • .cmtx — e-Sword Commentary
  • .dctx / .lexdbtx — e-Sword Dictionary or Lexicon
  • .mybible — MySword Bible

The converted module is saved into your modules folder automatically.

Note: Encrypted .bblx files (commercial modules) cannot be converted and will show an error.

Hiding modules

If you have many modules installed, you can hide individual ones from the picker dropdowns without deleting them. In the Library, click the eye icon next to any module to toggle its visibility. Hidden modules remain on disk but do not appear in navigation menus.

Renaming modules

The display name of any module is read from the description field in its info table. You can rename a module by editing this field in any SQLite editor (such as DB Browser for SQLite) — changes take effect the next time ScriptureStudy scans the folder.
